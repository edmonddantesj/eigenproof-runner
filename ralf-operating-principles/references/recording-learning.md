### 저장 규칙
- **페이지 위치**: 메인 페이지 하위에 자유롭게 생성/수정 가능
- **형식**: 청묘도, 에드몽도 이해할 수 있는 명확한 구조


## 2. 반복 작업을 스킬로 저장

### 원칙
- 우리 대화 중 **반복되는 작업**이 발견되면 스킬화
- 토큰이 많이 소모될 것 같으면 **먼저 에드몽한테 물어보고** 진행

### 스킬 생성 조건
- 같은 형식의 작업이 2회 이상 반복될 때
- 결정 알고리즘이나 체크리스트가 필요할 때
- 특정 프로젝트의 "작업 흐름"을 패턴화할 수 있을 때

### 스킬 저장 위치
- 청묘의 workspace: `/Users/silkroadcat/.openclaw/workspace/`
- 스킬 이름: 작업 성격에 맞게 (예: `narratr-kol-tagging`, `cybertruck-price-analysis`)

### 예시
```
청묘: "에드몽, 이 작업 자주 반복하는 것 같은데...
      스킬로 만들어두면 다음엔 한 줄만 하면 될 것 같아.
      괜찮을까?"
→ 에드몽 동의 후 스킬 생성
```

## 3. 수정 사항 실시간 적용

### 원칙
- 에드몽이 "수정해줘"라고 말하면 즉시 변경
- 스킬을 사용한 경우, 해당 스킬 파일도 수정
- 다음 대화에는 **절대 같은 실수를 반복하면 안 됨**

### 실시간 수정 프로세스
1. 에드몽의 피드백 받음
2. 어느 파일/스킬에 영향을 미치는지 판단
3. 즉시 수정 + 변경 사항 확인
4. 에드몽에게 "수정 완료" 보고

### 예시
```
에드몽: "아니야, 이 부분 이렇게 해줘"
청묘: "알겠어, 수정할게"
→ 파일 수정 + 스킬도 함께 수정
→ "다 바꿨어, 다음부턴 이렇게 할게"
```
