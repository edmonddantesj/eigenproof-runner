## 0) Executive summary
이 문서는 “요약 프롬프트”의 위험(논리 공백/편향 은닉)을 전제로, 문서를 AI로 다룰 때 **외부지식 개입을 차단하고, 주장과 증거를 분리해 검증하고, 최종 결론을 스스로 파괴적으로 재검토**하는 15개 모듈(Phase 0~5)로 구성된 ‘판단의 OS’를 제안한다. 핵심은 *먼저 부수고(검증), 살아남은 뼈대에서 신호를 추출*하는 순서다.

## 1) 전체 구조 (Phase 0~5)
### Phase 0 — 분석 시작 전 영점 조준
- Zero-base 선언: 업로드된 원문만 기준, 외부 지식 개입 금지, 기존 답변/요약에 얽매이지 않기.

### Phase 1 — 지식 장악 및 학습 (3 modules)
1) 파인만 해체기: 단순 설명 → 논리 공백 적발 → 공백 보완 재설명(비유는 원문 근거 필수)
2) 챕터 마스터리 맵: 3문장 개요 + 계층 구조 + 복습 체크리스트 + 난이도 높은 개념 3개
3) 역질문 인터뷰어: 실무 검증 질문 10개(원문 인용으로 교정)

### Phase 2 — 구조화 및 본질 추출 (3 modules)
4) 고순도 신호 추출기: 의사결정을 뒤집을 비가역 정보 3개(비교대상 없음 명시)
5) 무결성 리서치 브리프: 방법론/주장/한계 + 결론 vs 추측 분리
6) 타임라인 구조화: 가속 구간과 트리거 강조

### Phase 3 — 가혹한 검증 (4 modules)
7) 주장–증거 맵: 주장 | 원문 인용 | 증거 밀도 | 반대 증거
8) 모순 및 가정 추적기: 모순/숨은 가정 적발(원문 인용)
9) 논리적 공백 탐지기: 단절 지점 + 문서가 답 못하는 질문 10개
10) 취약점 스캐너: 방법론 결함/논리 비약/과장(원문 인용)

### Phase 4 — 실전 아웃풋 변환 (4 modules)
11) 의사결정 매트릭스: 옵션/기준/리스크/트레이드오프/철회 조건 (가중치/확신도% 금지)
12) SOP 및 실행 청사진: 전제/분기/실패 모드/즉시 중단 조건
13) 이해관계자 번역기: 경영진/엔지니어/실무자 관점 재구성
14) 실무 에셋 빌더: 블로그/기획안/10장 발표자료 뼈대(과장 금지, 원문 인용)

### Phase 5 — 메타 감 (1 module)
15) 자기 파괴적 최종 점검: 내부 모순, 치명적 누락 전제, 결론 무효화 외부지표 1개 정의
